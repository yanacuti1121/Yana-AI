## 📋 Proposed Features

### **HIGH PRIORITY (Weeks 1-2)**

#### **1. Circuit Breaker v2 (L5 Enhancement)**

**Current:** `token-budget-guard.sh` global circuit breaker (v1.6.0)
- Single CLOSED/OPEN/HALF_OPEN state
- Fixed cooldown: 60s → 300s → 1800s

**Upgrade:** Per-tool state tracking

**Implementation:**
- New file: Extend `core/hooks/token-budget-guard.sh`
- Data store: `.claude/state/per-tool-circuit.jsonl`
- Per-tool entries:
  ```json
  {
    "tool_name": "Bash",
    "state": "OPEN",
    "failure_count": 5,
    "last_failure_time": "2026-05-24T14:32:00Z",
    "cooldown_until": "2026-05-24T14:37:00Z",
    "backoff_exponent": 2
  }
  ```

**Features:**
- ✅ Per-tool failure tracking (track which tool keeps timing out)
- ✅ Adaptive backoff (exponential per tool)
- ✅ Fast-tier fallback (Claude Haiku for Bash, limit to 3 calls)
- ✅ Health check before HALF_OPEN transition
- ✅ Metrics export: `circuit-breaker-metrics.sh` (tool_name, state, recovery_time)

**Tests:** 6 test cases (per-tool open/close/fallback, cross-tool isolation)

**Metrics:**
- Time to implement: 2 days
- Lines of code: ~80 (core logic) + ~40 (test suite)
- Backward compat: ✅ (advisory by default, strict opt-in)

#### **4. Advanced Hook Composition**

**Problem:** Hooks fire independently; hard to enforce ordering

**Example:** What if `supply-chain-guard` blocks, but `prompt-injection-guard` wasn't checked yet?

**Solution:** Hook pipeline with exit code chaining

**Implementation:**
- New file: `core/scripts/compose-hooks.sh`
- Config: `core/config/hook-pipeline.json`

**Pipeline config:**
```json
{
  "pipelines": {
    "PreToolUse": [
      {
        "name": "tool-validator",
        "blocking": true,
        "timeout": 1000
      },
      {
        "name": "prompt-injection-guard",
        "blocking": true,
        "timeout": 1000
      },
      {
        "name": "supply-chain-guard",
        "blocking": true,
        "timeout": 1000
      },
      {
        "name": "token-budget-guard",
        "blocking": false,
        "timeout": 500
      }
    ]
  }
}
```

**Features:**
- ✅ Sequential execution with early exit on block
- ✅ Non-blocking warnings (collect, don't stop)
- ✅ Timeout per hook (fast fail)
- ✅ Hook health check (skip if unhealthy)

**Tests:** 3 test cases (pass through, early exit, timeout)

**Metrics:**
- Time: 1 day
- Lines: ~90
- Files: 2 new
