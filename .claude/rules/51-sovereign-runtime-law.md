# 51-sovereign-runtime-law

## Rule

All code generated by an agent at runtime MUST pass static AST analysis before execution. No agent-generated JavaScript, Python, or shell script may be loaded or executed without prior `sovereign-interceptor.js` clearance.

## Enforcement Layers

| Stage | Tool | Blocks |
|-------|------|--------|
| AST scan | `sovereign-interceptor.js scanCode()` | eval(), exec(), process.env, dynamic require(), child_process |
| Command allow-list | `sovereign-interceptor.js verifyCommand()` | Commands not in `allowedCommands` set |
| Honey-vault check | `sovereign-interceptor.js checkHoneyVault()` | Any payload touching canary env vars |
| Regex fallback | Built-in patterns | When acorn unavailable |

## Prohibited

- Hot-loading agent-generated code without AST pre-check
- Bypassing the interceptor via `Function()` constructor or `vm.runInNewContext()`
- Agent-generated scripts that access `process.env` directly
- Dynamic `require()` with non-literal string argument
- Code that references `__dirname` / `__filename` for path traversal

## Permitted Code Patterns (Agent Runtime)

- Pure computation functions with no I/O side effects
- Calls to pre-approved YAMTAM APIs via the tool proxy
- Static `import` declarations (resolved at parse time)

## Violation Response

1. Block execution — return exit code 3
2. Log AST violation with fingerprint to Merkle audit chain
3. Penalize generating agent's trust score (−15 points)
4. Quarantine agent if trust score drops below threshold
